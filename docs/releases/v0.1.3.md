## goddgs v0.1.3

Patch release focused on HTTP fingerprinting and response-handling correctness.

### Changes

- Added explicit response decompression handling for `gzip`, `br`, and `zstd` in transport.
- Fixed gzip decompression close-path bug by preserving original body handle.
- Added safe handling for comma-delimited `Content-Encoding` values.
- Updated Chrome `Accept-Encoding` profile to include `zstd`.
- Corrected script fetch hint to `Sec-Fetch-Site: same-site` for DDG subdomain flow.
- Added `Priority` header behavior for Chrome/Edge request profiles.
- Added decompression regression tests including noop and already-uncompressed cases.
